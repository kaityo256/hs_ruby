# 極小頂点被覆を列挙するサンプル

## 概要

与えられた入力に対して、極小頂点被覆(minimal hitting set)を列挙するサンプルコード。ここで列挙するのは極小(minimal)であって、最小(minimum)でないことに注意。極小頂点被覆は、他の頂点被覆を真部分集合として含まない頂点被覆のこと。最小頂点被覆は、頂点被覆の中で要素数が最小のもの。

## Usage 

    $ ruby hs.rb
    input
    01101010 : [7,6,4,2]
    11001010 : [8,7,4,2]
    11000110 : [8,7,3,2]
    00111010 : [6,5,4,2]
    01010101 : [7,5,3,1]
    11010100 : [8,7,5,3]
    11000101 : [8,7,3,1]
    00011110 : [5,4,3,2]
    01101001 : [7,6,4,1]
    00011011 : [5,4,2,1]
    hitting sets
    10000011 : [8,2,1]
    00000111 : [3,2,1]
    00100110 : [6,3,2]
    00010011 : [5,2,1]
    01000010 : [7,2]
    00001100 : [4,3]
    10001001 : [8,4,1]
    10011000 : [8,5,4]
    10100101 : [8,6,3,1]
    10110000 : [8,6,5]
    01001000 : [7,4]
    01010000 : [7,5]
    01100101 : [7,6,3,1]

最初に適当に作られた入力データ(デフォルトでは8bit中4ビットが立ったビット列が10個)が「input」として表示される。デバッグの都合でsrandしてあるので、毎回同じ結果が出るが、srandを外せば毎回異なる結果になる。このビット列すべてと論理積をとってゼロでないビット列を「hitting set」として列挙する。なお、速度などは考慮していない。
